01 サービスコンセプト 確定

サービス名： スナログ（snacklog）

一言コンセプト：
「初心者・女性でも安心して行ける」赤坂特化のスナック検索・口コミ・来店きっかけサイト。将来的には全国展開を想定した拡張可能な構造。

重要な定義：
「予約サイト」ではなく 「来店きっかけサイト」 と定義する。スナックの来店行動は流れ来店が主であり、予約コンバージョンは計測・期待できない。

参照モデル

  • 食べログ（網羅性・透明性）
  • エアビー（市場原理）
  • Airレジ / TableCheck（経営管理）
  • バイトル / ランサーズ（求人）

3者が使うプラットフォーム

来店ユーザー ／ 店舗オーナー・ママ ／ 求職者（働き手）

06 各ページ詳細仕様 作成中

TOP（ホーム） / — 最優先（MVP）

目的： 「今夜、赤坂でスナックを探している人」が直感的に検索を始められること

主要要素：

  • ヒーローエリア：大型検索バー（エリア・キーワード）
  • クイックフィルター：初めての方 / 一人飲み / カラオケあり / 女性歓迎
  • 今夜行けるスナック（即時来店フラグ付き店舗）
  • プレミアム掲載店スポットライト
  • 人気エリアリンク（赤坂見附・溜池山王）

CV導線： 検索実行 → 店舗詳細 → 電話 / SNS / 予約

店舗検索（一覧） /search — 最優先（MVP）

目的： ユーザーが条件を絞り込み、自分に合う店を発見する

絞り込み条件：

  • エリア（赤坂見附 / 溜池山王 / 赤坂）
  • 予算（〜¥5,000 / 〜¥10,000 / ¥10,000〜）
  • 設備タグ（カラオケ / ボトルキープ / カード可）
  • 雰囲気タグ（初心者歓迎 / 女性歓迎 / 一人歓迎）
  • 今夜行けるフラグ

店舗カード： 写真・店名・エリア・料金目安・口コミ評価・タグ

表示順： プレミアム上位固定 → スタンダード（マップピン）→ ベーシック

店舗詳細 /shops/:id — 最優先（MVP）

目的： 安心して来店できるよう十分な情報を提供し、CTAへ繋げる

主要要素：

  • 写真ギャラリー（3枚 / 10枚 / 無制限＋動画：プランによる）
  • 基本情報（住所・営業時間・定休日・地図）
  • 料金システム（セット料金 / チャージ / ボトルキープ）
  • ママ紹介（写真＋コメント）※スタンダード以上
  • タグ一覧（雰囲気・設備）
  • 口コミ一覧・投稿フォーム
  • 類似店舗（横スライダー）
  • 求人情報（掲載している場合のみ）

CTA（固定）： 📞 電話する / 💬 SNSで問い合わせ / 📅 予約する

口コミ投稿 /shops/:id/review — Phase2

目的： 来店証明コンテンツを蓄積し、次のユーザーの意思決定を支援する

入力項目：

  • 総合評価（★1〜5）
  • 雰囲気・料金・ママの対応（各★）
  • テキストコメント（任意）
  • 来店日・人数・利用シーン

求人（一覧・詳細・応募） /jobs — Phase2

目的： スナックで働きたい人と店舗をマッチングする

一覧の要素： 店名・エリア・時給・シフト・応募条件・写真

制限： プレミアムプランの店舗のみ掲載（検討中：論点A-3）

店舗管理ダッシュボード /manage — 最優先（MVP）

目的： 店舗オーナーが掲載情報を管理し、集客効果を確認する

  • 店舗情報編集（写真・基本情報・料金・SNS）
  • CVレポート（電話・SNSタップ数 ※スタンダード以上）
  • 予約管理（団体予約の確認・承認 ※プレミアム）
  • 求人管理・応募者確認（※プレミアム）
  • BIダッシュボード：売上・来客推移（※プレミアム）

プラン比較ページ /pricing — Phase2

目的： 店舗オーナーが自分に合ったプランを選べるよう、機能・価格を明確に比較提示する

  • 3プラン横並び比較表（ベーシック / スタンダード / プレミアム）
  • 機能マトリクス（○ / — で一覧）
  • 各プランCTAボタン
  • よくある質問（FAQ）

07 店舗詳細ページ CTA仕様 確定

CTAの種類（3種）：

  • 📞 電話する — 常時表示
  • 💬 SNSで問い合わせ — 登録済みのみ表示
  • 📅 予約する — 団体向け・URL登録時のみ

表示優先順位： ① 電話する（最優先・常時）→ ② SNS（display_order順）→ ③ 予約する（任意）

「予約する」の文言は一人客向け「空席確認する」、団体向け「席を確保する（2名以上）」への出し分けを検討中（論点C-3）。

10 UI / UX方針 確定

  • スマホファースト（レスポンシブ）
  • TOPは来店ユーザー体験を最優先
  • 全国展開を前提とした拡張可能な構造
  • 店舗・求人向け入口はメニューに集約

初心者でも入れるよう、料金の透明性・写真・ママ紹介を前面に出す。「今夜行けるスナック」フラグで即時来店を促す。

11 競合分析・差別化 確定

競合弱点
スナックガイド情報が古くUXが弱い
ナイトスタイルキャバクラ中心でスナック専門性なし
ホットペッパーグルメスナックカテゴリが弱い
Googleマップ口コミ中心で体系化されていない
スナックナビ月6万セッション・SNSとの体系化なし

差別化の核： 「信頼性の高い情報 ＋ 来店きっかけの導線 ＋ 安心×雰囲気」を統合。
